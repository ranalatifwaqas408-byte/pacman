1. Abstract
This report presents the design and implementation of a simplified, text-based Pac-Man game using Java. The game features a hero character who navigates a maze, collects pellets, and avoids an enemy ghost. The primary focus is on demonstrating core Object-Oriented Programming (OOP) principles including encapsulation, modularity, and the effective use of Java Collections. The application runs in a console environment and provides real-time keyboard input for player movement.

2. System Architecture
The system follows a modular architecture with five primary classes, each handling a distinct responsibility. The architecture is event-driven, where user input triggers movement, collision detection, and game state updates.

3. Detailed Class Design
3.1 PacMan (Main Driver)
This class contains the game loop, which continues until either the player collides with the ghost or all pellets are collected.

Key Logic Flow:

java
while (true) {
    maze.render(player, ghost, pellets);  // Display game state
    char key = input.next().charAt(0);    // Get user input
    player.movePlayer(key, maze);          // Update player position
    pellets.consume(...);                  // Check for pellet collection
    ghost.randomMove(maze);                // Move enemy
    // Collision & win checks
}
Design Choice: The main class acts solely as an orchestrator, delegating all domain logic to specialized classes.

3.2 Maze Class
The maze is stored as a 10×10 2D character array. The layout is hardcoded using a string array for simplicity and reproducibility.

Collision Detection:

java
public boolean blocked(int x, int y) {
    if (x < 0 || y < 0 || x >= r || y >= c) return true;
    return grid[x][y] == '#';
}
Rendering Logic:

C → Hero (Pac-Man)

X → Ghost (Enemy)

# → Wall

. → Pellet (disappears when consumed)

 (space) → Empty path

3.4 Enemy Class
The enemy implements simple random movement AI. At each turn, it randomly selects one of four directions (up, down, left, right). If the chosen direction leads to a wall, the ghost stays in place for that turn.

Movement Directions:

3.5 Pellet Class (Using HashSet)
This class demonstrates efficient use of Java Collections. Pellets are stored as string keys in a HashSet<String>.

Why HashSet?

O(1) average time complexity for add, remove, and contains operations

Automatic duplicate prevention

Simple key generation: row + "-" + col

Key Methods:

java
public void place(int x, int y) { items.add(x + "-" + y); }
public boolean consume(int x, int y) { return items.remove(x + "-" + y); }
public boolean exists(int x, int y) { return items.contains(x + "-" + y); }
public boolean finished() { return items.isEmpty(); }

8. OOP Principles Demonstrated
Principle	Implementation
Encapsulation	All class fields are private with public getters
Abstraction	Each class hides its internal logic (e.g., Pellet hides HashSet implementation)
Single Responsibility	Maze handles rendering, Hero handles movement, Pellet handles collection
Separation of Concerns	Game logic, rendering, and data storage are in separate classes

9. Conclusion
This project successfully implements a functional Pac-Man style game using core Java and Object-Oriented Programming principles. The architecture separates concerns effectively, making the code maintainable and extensible. The use of HashSet for pellet management demonstrates efficient data structure selection. While the current implementation is console-based with simple random ghost movement, it provides a solid foundation for future enhancements such as intelligent ghost AI, multiple levels, and graphical interfaces.
